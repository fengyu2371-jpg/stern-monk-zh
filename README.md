# stern-monk-zh-tw v26｜今日穿搭推薦版

這是修士 Bot 的 Railway 部署精簡版。

## 部署檔案

```text
main.py
requirements.txt
railway.toml
data/
  faq_zh_tw.json
  tutorials_zh_tw.json
  dialogue.json
```

程式碼只保留 `main.py` 一份；`data/` 是教學與台詞資料。

## 指令

```text
/學生資料
/今日穿搭推薦
/修士狀態
```

原本的 `/建立修士面板` 已移除。

## 主要規則

- `/學生資料` 會直接公開顯示使用者自己的學生資料。
- 其他玩家看得到，但不能操作按鈕。
- 其他玩家點擊時只收到私密提示，不會改動別人的資料。
- 本人每次有效操作都會重新計時。
- 5 分鐘沒有操作後，按鈕與選單會被移除，公開資料仍保留。
- 要再次操作，重新輸入 `/學生資料`。
- 我的地點採總覽＋下拉選單管理，地點可刪除，刪除前有二次確認。
- 商店街會讀取所有公開店鋪，並顯示店主／經營者。


## v24 地點編輯

在「我的地點」選擇單一地點後，可按：

```text
✏️ 編輯地點
```

可修改：

- 地點名稱
- 店主／經營者／居住者
- 所在區域
- 目前狀態
- 地點簡介

地點類型、來源與公開狀態不放在編輯表單中：
公開狀態仍使用原本的「公開顯示」按鈕調整。
其他玩家仍只能查看，不能編輯別人的地點。


## v25 全畫面五分鐘鎖定

所有由 `/學生資料` 開啟的個人操作頁都共用同一個五分鐘計時器。

每次本人有效操作後重新計時。超過五分鐘沒有互動時，不論目前停在學生資料、地點管理、編輯表單、刪除確認或神諭頁，原訊息都會切換成：

```text
🔒 操作畫面已鎖定

這位學生的操作畫面已超過 5 分鐘沒有互動。
為避免他人誤觸，所有按鈕與選單已關閉。
需要繼續操作時，請重新輸入 /學生資料。
```

鎖定後不提供解鎖按鈕。


## v26 今日穿搭推薦

新增一般成員指令：

```text
/今日穿搭推薦
```

流程：

1. 選擇穿搭方向：男裝、女裝、無性別穿搭、隨機。
2. 輸入今日關鍵詞。
3. 呼叫 OpenAI 產生推薦。
4. 以 Discord Embed 回傳。

限制與安全規則：

- 每位玩家每天只能使用 1 次。
- 使用既有 SQLite `usage_counters` 紀錄，不新增或修改神父玩家數值。
- 不把 Discord 顯示名稱、學生姓名或 user_id 放進 OpenAI prompt。
- 不假設玩家性別、身材、年齡、真實身分或預算。
- 不輸出色情、過度暴露、身材曲線描述或昂貴品牌指定。
- API 失敗或 JSON 解析失敗時，會改用本地 fallback 穿搭，不讓指令報錯。
